Résumé du projet : Assistant IA de support e-commerce (Local)
Objectif

Développer un assistant IA de support e-commerce fonctionnant entièrement en local, capable d'interagir avec les clients via un chatbot.

L'assistant devra comprendre les demandes en langage naturel, interagir avec la base de données de la boutique et effectuer des actions comme :

Vérifier la disponibilité d'un produit
Rechercher un produit même si son nom est approximatif
Créer une commande
Suivre une commande
Répondre aux questions générales
Demander des précisions lorsque nécessaire

L'objectif est de construire une V1 simple, propre et évolutive.

Philosophie du projet

Le modèle IA ne prend jamais de décisions métier.

Il est uniquement responsable de :

comprendre le message du client ;
identifier son intention ;
extraire les informations importantes.

Toute la logique métier reste dans Django.

Rôle de l'IA

Par exemple :

Client :

Je voudrais deux Samsung S25.

L'IA renvoie uniquement :

{
    "intent":"create_order",
    "entities":{
        "product":"Samsung S25",
        "quantity":2
    }
}

Ensuite Django :

recherche le produit ;
vérifie le stock ;
crée la commande ;
enregistre les données ;
construit la réponse.

Le backend communiquera avec Ollama via son API HTTP locale.

POST http://ollama:11434/api/chat

Aucune commande shell ne sera utilisée.

Objectif final

Construire un assistant IA e-commerce intelligent, entièrement local, sécurisé et extensible, où :

Ollama (Qwen3:4b) comprend les demandes des utilisateurs et extrait les intentions.
Django exécute toute la logique métier, interagit avec PostgreSQL et garantit la cohérence des données.
Bootstrap + JavaScript offrent une interface de chat simple, réactive et facile à maintenir.
Docker Compose permet de lancer tout l'environnement avec une seule commande.
